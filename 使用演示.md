## 🌐 Web 版本演示

### 启动 Web 服务器
```bash
python web_monitor.py
```

### 输出
```
Web 监控服务器启动中...
访问地址: http://127.0.0.1:5000
按 Ctrl+C 停止服务器
 * Serving Flask app 'web_monitor'
 * Debug mode: off
 * Running on http://127.0.0.1:5000
```

### Web 界面特点

#### 1. 主界面
- **顶部标题**：显示当前时间和在线状态指示器
- **卡片式布局**：CPU、内存、网络信息分别显示在独立卡片中
- **渐变色设计**：紫色到粉色的渐变背景
- **响应式**：自动适配不同屏幕尺寸

#### 2. CPU 卡片
- 显示总体使用率和进度条
- 显示核心数和频率
- 展开显示每个核心的独立使用率

#### 3. 内存卡片
- 显示内存使用率和进度条
- 显示总内存、已使用、可用容量
- 人性化单位显示（GB/MB/KB）

#### 4. 网络卡片
- 实时上传/下载速度
- 总流量统计
- 活动连接数

#### 5. 磁盘信息区域
- 列表显示所有分区
- 每个分区显示：设备名、挂载点、文件系统
- 容量和使用率可视化

#### 6. 实时图表
- 使用 Chart.js 绘制曲线图
- 显示最近 60 秒的 CPU 和内存使用率
- 平滑动画，实时更新

#### 7. 进程列表
- 表格形式显示 Top 10 进程
- 按 CPU 使用率排序
- 显示 PID、进程名、CPU%、内存%、状态

## 📱 移动端访问

### 在局域网内访问

1. 启动服务器（允许外部访问）：
```bash
python web_monitor.py --host 0.0.0.0
```

2. 查看本机 IP 地址：
```bash
# Windows
ipconfig

# Linux/Mac
ifconfig
```

3. 在手机浏览器访问：
```
http://192.168.1.100:5000
（替换为你的实际 IP）
```

## 🔧 API 使用示例

```bash
curl http://127.0.0.1:5000/api/system
```

### 返回数据格式（JSON）
```json
{
  "timestamp": "2026-06-05T18:19:12.847214",
  "cpu": {
    "percent": 7.4,
    "count": 20,
    "frequency": 2300.0,
    "per_core": [0.0, 0.0, 0.0, 37.5, ...]
  },
  "memory": {
    "total": 16849293312,
    "used": 14079082496,
    "percent": 83.6,
    "total_readable": "15.69 GB",
    "used_readable": "13.11 GB"
  },
  "disk": {...},
  "network": {...},
  "processes": [...]
}
```

### 获取历史数据
```bash
curl http://127.0.0.1:5000/api/history
```

### 在其他程序中调用
```python
import requests

response = requests.get('http://127.0.0.1:5000/api/system')
data = response.json()

print(f"CPU 使用率: {data['cpu']['percent']}%")
print(f"内存使用率: {data['memory']['percent']}%")
```

## 💡 使用技巧

### 1. 自定义刷新频率
```bash
# 更快的刷新（1秒）
python system_monitor.py -i 1

# 更慢的刷新（5秒，节省资源）
python system_monitor.py -i 5
```

### 2. 保存定期快照
创建一个脚本定期保存快照：
```bash
# Linux/Mac
while true; do
  python system_monitor.py -s -o "snapshot_$(date +%Y%m%d_%H%M%S).json"
  sleep 300  # 每5分钟保存一次
done
```

### 3. 监控特定指标
修改代码只显示关心的指标，例如只监控 CPU 和内存。

### 4. 远程监控
通过 SSH 隧道安全访问远程服务器的 Web 界面：
```bash
ssh -L 5000:localhost:5000 user@remote-server
```
然后在本地访问 http://localhost:5000

## 🎯 实际应用场景

### 场景 1：性能测试
在运行性能测试时，同时启动监控工具观察资源使用情况。

### 场景 2：故障排查
系统变慢时，快速查看是哪个进程占用了大量 CPU 或内存。

### 场景 3：资源规划
长期监控并保存快照，分析历史数据来规划资源升级。

### 场景 4：开发调试
开发时监控应用的资源占用，优化性能。

### 场景 5：服务器监控
在服务器上运行命令行版本，或通过 Web 界面远程监控。

---

## ⚠️ 注意事项

1. **权限**：某些进程信息可能需要管理员/root 权限
2. **性能**：刷新间隔太小（<1秒）可能增加系统负担
3. **网络安全**：使用 `--host 0.0.0.0` 时注意网络安全
4. **编码**：Windows 终端如果乱码，运行 `chcp 65001` 设置 UTF-8
5. **防火墙**：Web 版本可能需要在防火墙中开放端口
