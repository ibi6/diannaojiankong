# 项目文件说明

## 📁 文件结构

```
电脑状态监控工具/
├── system_monitor.py          # 命令行版本监控程序
├── web_monitor.py             # Web 版本监控服务器
├── requirements.txt           # Python 依赖列表
├── README.md                  # 项目文档
├── 启动监控.bat               # Windows 快速启动脚本（命令行版）
├── 启动Web监控.bat            # Windows 快速启动脚本（Web版）
├── 保存快照.bat               # Windows 快速保存快照脚本
└── templates/
    └── index.html             # Web 界面 HTML 页面
```

## 📝 文件详细说明

### system_monitor.py
**命令行版本的系统监控工具**

- 功能：在终端中实时显示系统状态
- 特点：轻量级、无需浏览器、适合服务器环境
- 输出：彩色终端界面，带进度条显示
- 退出时自动保存快照到 JSON 文件

### web_monitor.py
**Web 版本的监控服务器**

- 功能：启动 Flask Web 服务器，提供可视化界面
- 特点：图形化界面、实时图表、支持多设备访问
- 端口：默认 5000，可自定义
- API：提供 RESTful API 供其他程序调用
- 后台线程：自动收集历史数据（最近 60 个数据点）

### templates/index.html
**Web 前端界面**

- 技术栈：HTML5 + CSS3 + JavaScript + Chart.js
- 设计：现代化渐变色界面，卡片式布局
- 响应式：支持桌面、平板、手机
- 实时更新：每 2 秒通过 Ajax 获取最新数据
- 图表：使用 Chart.js 显示 CPU 和内存历史曲线

### requirements.txt
**Python 依赖清单**

```
psutil>=5.9.0        # 系统信息采集库
flask>=2.3.0         # Web 框架
flask-cors>=4.0.0    # 跨域资源共享支持
```

### 启动脚本（.bat 文件）

- **启动监控.bat**：启动命令行版本
- **启动Web监控.bat**：启动 Web 服务器
- **保存快照.bat**：快速保存系统状态快照

这些脚本会自动设置 UTF-8 编码，避免中文乱码问题。

## 🔧 技术架构

### 后端技术
- **Python 3.6+**：主要开发语言
- **psutil**：跨平台系统信息采集库
- **Flask**：轻量级 Web 框架
- **Flask-CORS**：处理跨域请求
- **threading**：多线程支持（后台数据采集）
- **collections.deque**：高效的历史数据存储

### 前端技术
- **HTML5**：页面结构
- **CSS3**：样式和动画（渐变、过渡效果）
- **JavaScript ES6+**：交互逻辑
- **Chart.js 4.x**：图表库（实时曲线图）
- **Fetch API**：Ajax 数据请求

## 📊 监控指标

### CPU
- 总体使用率
- 各核心独立使用率
- CPU 核心数量
- 当前频率（MHz）

### 内存
- 总内存容量
- 已使用内存
- 可用内存
- 使用百分比
- 虚拟内存（交换空间）

### 磁盘
- 各分区信息（设备、挂载点、文件系统）
- 总容量、已使用、剩余空间
- 使用百分比
- 实时读写速度

### 网络
- 上传/下载速度（实时）
- 总上传/下载流量（累计）
- 活动网络连接数

### 进程
- 进程 PID
- 进程名称
- CPU 使用率
- 内存使用率
- 进程状态

## 🚀 性能特点

### 资源占用
- CPU：< 1%（空闲时）
- 内存：~30-50 MB
- 刷新频率：可自定义（默认 2 秒）

### 数据采集
- 使用 psutil 高效 API
- 增量计算网络和磁盘速度
- 历史数据使用 deque 自动限制大小

### Web 性能
- 异步数据采集（独立线程）
- 前端仅更新变化的数据
- Chart.js 使用 Canvas 硬件加速

## 🔒 权限说明

### Windows
- 基本功能：无需特殊权限
- 完整进程信息：建议以管理员身份运行

### Linux/macOS
- 基本功能：普通用户即可
- 完整进程信息：可能需要 sudo

## 🌐 网络安全

### 默认配置（安全）
- 监听地址：127.0.0.1（仅本机可访问）
- 端口：5000

### 局域网访问
- 使用 `--host 0.0.0.0` 允许外部访问
- 建议在受信任的网络环境使用
- 无身份验证机制，不推荐公网暴露

## 🎯 适用场景

### 命令行版本适合
- 服务器环境监控
- SSH 远程登录
- 资源占用排查
- 自动化脚本集成

### Web 版本适合
- 桌面电脑监控
- 团队共享监控
- 多设备访问需求
- 需要历史数据可视化

## 📖 扩展建议

如果想进一步扩展功能，可以考虑：

1. **数据持久化**：将历史数据保存到数据库（SQLite、InfluxDB）
2. **告警功能**：CPU/内存超过阈值时发送通知
3. **用户认证**：添加登录功能保护监控数据
4. **多机监控**：支持监控多台服务器
5. **性能分析**：添加更详细的性能指标（GPU、温度等）
6. **导出报告**：生成 PDF/Excel 性能报告
7. **WebSocket**：使用 WebSocket 替代轮询，降低延迟
